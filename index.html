§1 Introducción a OCL

1.1 ¿Qué es OCL?

OCL (Object Constraint Language) es un lenguaje formal estándar definido por la OMG (Object Management Group) que permite expresar restricciones y reglas de negocio sobre modelos UML de forma precisa e inequívoca.

Característica Descripción
Lenguaje formal Semántica matemáticamente bien definida, sin ambigüedad
Parte de UML Complementa los diagramas UML; estandarizado en UML 2.x
Declarativo Describe qué debe cumplirse, no cómo lograrlo
Sin efectos laterales Las expresiones OCL no modifican el estado del modelo
Tipado fuerte Cada expresión tiene un tipo estático verificable

1.2 ¿Por qué necesitamos OCL?

Los diagramas UML son herramientas visuales muy útiles, pero tienen limitaciones expresivas intrínsecas. Ciertos requisitos del dominio sencillamente no pueden representarse con notación gráfica:

⚠
Limitación de UML sin OCL

Un diagrama de clases puede mostrar que un Flight tiene una lista de passengers y que un Plane tiene un atributo capacity, pero no puede expresar que el número de pasajeros nunca debe superar la capacidad del avión.

Con OCL, esa regla se expresa de forma concisa y sin ambigüedad:

OCL 
context Flight
inv pasajerosValidos:
    self.passengers->size() <= self.plane.capacity
✓
La invariante siempre se cumple

Esta restricción debe ser verdadera para toda instancia de Flight en todo momento del ciclo de vida del objeto. El evaluador OCL puede verificarlo automáticamente.

1.3 Usos de OCL

OCL se emplea en seis contextos fundamentales dentro de un modelo UML:

  • inv
    Invariantes de clase (inv)
    Condiciones booleanas que deben cumplirse para toda instancia de la clase, en cualquier estado estable del sistema.
  • init
    Valores iniciales de atributos (init)
    Definen el valor que toma un atributo en el momento de creación de la instancia.
  • der
    Atributos derivados (derive)
    Especifican cómo calcular el valor de un atributo marcado con / en el diagrama de clases UML.
  • bod
    Cuerpo de operaciones de consulta (body)
    Proporcionan la implementación declarativa de una operación de solo lectura (que no modifica estado).
  • pre
    Precondiciones de operaciones (pre)
    Condiciones que deben ser verdaderas antes de invocar una operación; responsabilidad del llamador.
  • pos
    Postcondiciones de operaciones (post)
    Condiciones que deben ser verdaderas después de ejecutar la operación; responsabilidad del proveedor. Admiten result y @pre.

§2 Sintaxis Básica

Toda expresión OCL comienza con una cláusula context que indica la clase u operación a la que se aplica la restricción, seguida de una estereotipo (inv, init, derive, body, pre o post) y la expresión propiamente dicha.

Estereotipo Aplicado a Propósito Tipo de expresión
inv Clase / tipo Invariante de integridad Booleana
init Atributo Valor inicial al crear instancia Mismo tipo que el atributo
derive Atributo derivado Regla de cálculo del atributo Mismo tipo que el atributo
body Operación de consulta Implementación declarativa Tipo de retorno de la operación
pre Operación Condición previa Booleana
post Operación Condición posterior Booleana (puede usar result, @pre)

2.1 Invariantes inv

Una invariante es una expresión booleana que debe ser true para toda instancia de la clase, en todo momento estable (antes y después de cualquier operación).

Sintaxis
context NombreClase
inv [nombre_opcional]: <expresión_booleana>
ℹ
El rol de self

Dentro de una invariante, self hace referencia a la instancia concreta que está siendo evaluada. Es equivalente al this de lenguajes orientados a objetos.

Ejemplo

OCL — Invariantes 
-- El cliente debe ser mayor de edad
context Customer
inv mayorDeEdad: self.age >= 18

-- La fecha de inicio debe ser anterior a la de vencimiento
context CustomerCard
inv fechasValidas: self.validFrom < self.goodThru

2.2 Valores Iniciales init

Especifica el valor inicial que toma un atributo en el momento en que se crea una nueva instancia de la clase. Debe respetar el tipo declarado del atributo.

Sintaxis
context Clase::atributo : Tipo
init: <expresión>

Ejemplo

OCL — Valor inicial 
-- Al crear una cuenta, el saldo de puntos comienza en cero
context LoyaltyAccount::points : Integer
init: 0

2.3 Reglas de Derivación derive

Describe cómo calcular el valor de un atributo derivado (marcado con / en el diagrama de clases UML). El atributo no se almacena; su valor se recalcula cada vez que se accede.

Sintaxis
context Clase::atributo : Tipo
derive: <expresión>

Ejemplo

OCL — Atributo derivado 
-- El nombre impreso combina tratamiento y apellido del titular
context CustomerCard::printedName : String
derive: owner.title.concat(' ').concat(owner.lastName)

2.4 Cuerpo de Operación body

Proporciona la implementación declarativa de una operación de consulta: una operación que retorna un valor y no modifica el estado del objeto (sin efectos laterales).

Sintaxis
context Clase::operacion(param: Tipo) : TipoRetorno
body: <expresión>

Ejemplo

OCL — Body de consulta 
-- Retorna todos los servicios de un nivel dado
context LoyaltyProgram::getServices(k: ServiceLevel) : Set(Service)
body: partners->collect(p | p.deliveredServices)
          ->select(s | s.serviceLevel = k)

2.5 Precondiciones pre

Condición que debe ser true antes de invocar la operación. Es responsabilidad del llamador garantizarla. Si la precondición se viola, el comportamiento de la operación es indefinido.

Sintaxis
context Clase::operacion(param: Tipo)
pre [nombre_opcional]: <expresión_booleana>

Ejemplo

OCL — Precondición 
-- Solo se puede inscribir un cliente que aún no está en el programa
context LoyaltyProgram::enroll(c: Customer)
pre clienteNoInscrito: not self.participants->includes(c)

2.6 Postcondiciones post

Condición que debe ser true después de ejecutar la operación. Es responsabilidad del proveedor (la operación misma) garantizarla. Permiten usar dos operadores especiales:

result
Hace referencia al valor que retorna la operación.
@pre
Se aplica a un atributo o colección para obtener su valor antes de que la operación se ejecutara. Sintaxis: atributo@pre.
Sintaxis
context Clase::operacion(param: Tipo) : Tipo
post [nombre_opcional]: <expresión_booleana>

Ejemplo

OCL — Postcondiciones 
-- Tras enroll, el cliente está en la lista y el tamaño aumentó en 1
context LoyaltyProgram::enroll(c: Customer)
post inscripto:
    self.participants->includes(c)
post sinCambiosTamanio:
    self.participants->size() = self.participants@pre->size() + 1
★
Palabras clave especiales en postcondiciones

result — El valor que devuelve la operación. Solo disponible en post de operaciones con tipo de retorno.

atributo@pre — El valor que tenía atributo justo antes de que se ejecutara la operación. Fundamental para expresar cómo cambia el estado: por ejemplo, self.points@pre + earned verifica que los puntos se incrementaron en la cantidad correcta.

§3 Tipos OCL

3.1 Tipos Primitivos

Tipo Descripción Ejemplos de valores Operaciones clave
Integer Número entero 0, -5, 42 +, -, *, /, abs(), mod(), div(), max(), min()
Real Número real 3.14, -0.5 +, -, *, /, abs(), floor(), ceil(), round()
String Cadena de texto 'hola', 'OCL' size(), concat(), substring(), toUpper(), toLower(), indexOf()
Boolean Valor lógico true, false and, or, not, xor, implies, if-then-else
UnlimitedNatural Natural + infinito 0, 1, * Usado en multiplicidades

3.2 Tipos Colección

Tipo Ordenado Duplicados Descripción
Set(T) No No Conjunto matemático. El más común.

Sequence(T) Sí Sí Lista ordenada con duplicados.
OrderedSet(T) Sí No Conjunto ordenado sin duplicados.
La navegación por asociaciones devuelve Set si la asociación no es ordenada. collect() siempre devuelve Bag.

3.3 Operaciones de String

Operación Descripción

substring(i, j) Subcadena desde el índice i hasta j (1-based)

toLowerCase() Convierte a minúsculas
indexOf(s) Posición de la primera ocurrencia de s (0 si no existe)
includes(s) True si la cadena contiene la subcadena s

3.4 Operaciones de Boolean

Operación / Operador Descripción Ejemplo
and Conjunción lógica a and b
or Disyunción lógica a or b
not Negación lógica not a
xor Disyunción exclusiva a xor b
implies Implicación lógica: (not a) or b a implies b

implies: a implies b es equivalente a (not a) or b. Solo es false cuando a = true y b = false. En todos los demás casos es true.

§4 Operaciones de Colección

Las operaciones de colección se aplican con el operador ->. La mayoría acepta una expresión iteradora con la sintaxis coleccion->operacion(x | expresion_sobre_x).

4.1 Cardinalidad y Existencia

Operación Tipo retorno Descripción Ejemplo
->size() Integer Número de elementos self.employees->size()
->isEmpty() Boolean True si la colección está vacía self.accounts->isEmpty()
->notEmpty() Boolean True si la colección no está vacía self.orders->notEmpty()
->count(obj) Integer Número de ocurrencias de obj self.items->count(item)

4.2 Pertenencia

Operación Tipo retorno Descripción
->includes(obj) Boolean True si obj está en la colección
->excludes(obj) Boolean True si obj NO está en la colección
->includesAll(col) Boolean True si contiene todos los elementos de col
->excludesAll(col) Boolean True si no contiene ningún elemento de col

4.3 Filtrado

Operación Descripción
->select(x | condición) Subcolección con los elementos que cumplen la condición
->reject(x | condición) Subcolección con los elementos que no cumplen la condición 
-- Empleados mayores de 30
self.employees->select(e | e.age > 30)

-- Cuentas con saldo negativo
self.accounts->reject(a | a.balance >= 0)

4.4 Transformación

Operación Descripción
->collect(x | expresión) Nueva colección (Bag) aplicando la expresión a cada elemento
->flatten() Aplana colecciones de colecciones en una sola colección
collect() siempre retorna un Bag. Si se necesita un Set, usar ->collect(...)->asSet(). Atajo: self.role.propiedad equivale a self.role->collect(propiedad). 
-- Nombres de todos los empleados
self.employees->collect(e | e.name)
-- equivalente a:
self.employees.name

-- Edades únicas de empleados
self.employees->collect(e | e.age)->asSet()

4.5 Cuantificadores

Operación Descripción
->forAll(x | condición) True si todos los elementos cumplen la condición
->exists(x | condición) True si al menos uno cumple la condición
->one(x | condición) True si exactamente uno cumple la condición
->isUnique(x | expresión) True si todos los valores de la expresión son distintos entre sí
->any(x | condición) Retorna un elemento que cumple la condición (resultado indeterminado) 
-- Todos los empleados son mayores de edad
self.employees->forAll(e | e.age >= 18)

-- Existe al menos un empleado de IT
self.employees->exists(e | e.department = 'IT')

-- Exactamente una cuenta está activa
self.accounts->one(a | a.active = true)

-- Todos los nombres de empleados son únicos
self.employees->isUnique(e | e.name)

-- forAll con dos variables (pares)
self.employees->forAll(e1, e2 | e1 <> e2 implies e1.name <> e2.name)

4.6 Agregación

Operación Descripción
->sum() Suma de todos los elementos numéricos
->min() Elemento mínimo
->max() Elemento máximo
->sortedBy(x | expr) Retorna una Sequence ordenada según la expresión expr 
-- Suma de puntos de todas las cuentas
self.accounts->collect(a | a.points)->sum()

-- Empleados ordenados por salario
self.employees->sortedBy(e | e.salary)

4.7 Operaciones de Conjunto

Operación Descripción
->union(col) Unión de las dos colecciones
->intersection(col) Intersección: elementos comunes a ambas
->including(obj) Nueva colección añadiendo obj
->excluding(obj) Nueva colección eliminando todas las ocurrencias de obj
->symmetricDifference(col) Diferencia simétrica — solo disponible en Set

4.8 Conversión entre Colecciones

Operación Descripción
->asSet() Convierte a Set (elimina duplicados)
->asBag() Convierte a Bag
->asSequence() Convierte a Sequence
->asOrderedSet() Convierte a OrderedSet
Usar ->asSet()->size() = col->size() para verificar que una colección no contiene duplicados.

§5 Navegación

5.1 Operadores . y ->

Operador Aplicado a Uso Ejemplo
. Objeto individual Acceder a una propiedad o llamar a una operación de objeto self.age, self.name.toUpperCase()
-> Colección Aplicar una operación de colección self.employees->size()
-> Objeto individual Auto-eleva el objeto a Set{obj} y aplica la operación de colección self->includes(x)

5.2 Navegación por Asociaciones

Se navega usando el nombre del rol en el extremo opuesto de la asociación. 

-- Si Person tiene asociación con Company (rol: employer)
context Person
inv: self.employer.name <> ''

-- Si Company tiene asociación con Person (rol: employees, multiplicidad *)
context Company
inv: self.employees->size() > 0
Cuando la multiplicidad es 0..1 o 1, la navegación devuelve un objeto (o null). Cuando es *, devuelve una colección.

5.3 Atajo de Navegación con collect

-- Estas dos expresiones son equivalentes:
self.employees.name
self.employees->collect(e | e.name)

-- Y estas también:
self.flights.passengers.name
self.flights->collect(f | f.passengers)->collect(p | p.name)

5.4 Navegación desde/hacia Clase Asociación

-- Navegando HACIA la clase asociación (usar nombre de rol de la clase asoc.)
-- Si Marriage es clase asociación entre Person (husband/wife)
context Person
inv: self.Marriage->notEmpty() implies self.age > 16

-- Navegando DESDE la clase asociación
context Marriage
inv: self.husband.age > 16 and self.wife.age > 16

5.5 Operaciones OCL sobre Objetos

Operación Descripción
oclIsTypeOf(T) True si el objeto es exactamente del tipo T (no subtipos)
oclIsKindOf(T) True si el objeto es del tipo T o cualquier subtipo de T
oclAsType(T) Cast: trata el objeto como tipo T, dando acceso a sus propiedades
oclIsUndefined() True si el objeto es null o no está definido
oclIsNew() True si el objeto fue creado durante esta operación (solo válido en postcondiciones) 
-- Verificar tipo exacto
self.pet.oclIsTypeOf(Dog)  -- true solo si es Dog, no Poodle

-- Verificar tipo o subtipo
self.pet.oclIsKindOf(Animal)  -- true para Dog, Cat, etc.

-- Cast para acceder a propiedades de subtipo
self.oclAsType(Employee).salary

-- Acceder a propiedad redefinida de superclase
self.oclAsType(Person).age

5.6 allInstances() y Propiedades Estáticas

-- Todas las instancias de una clase
context Person
inv nombresUnicos: Person.allInstances()->isUnique(p | p.name)

-- Acceso a atributo/operación estática
context MyClass
inv: MyClass::maxInstances > 0

5.7 Expresión if-then-else

if <condición> then
    <expresión_si_verdadero>
else
    <expresión_si_falso>
endif

-- Ejemplo:
context Person
inv: if self.age < 18 then
         self.income < 100
     else
         self.income < 200
     endif

5.8 Expresión let

let <variable> : <Tipo> = <expresión>
in <expresión_que_usa_variable>

-- Ejemplo:
context Person
inv salarioRazonable:
    let ingresoAnual : Real = self.income * 12
    in ingresoAnual < 1000000

-- Puede encadenarse:
let x : Integer = self.value * 2
in let y : Integer = x + 10
   in y > 0

5.9 Tuplas

-- Definir una tupla
Tuple{nombre: String = 'Juan', edad: Integer = 30}

-- Acceder a campos
miTupla.nombre

-- Colección de tuplas (ejemplo común)
context Person
let estadisticas : Set(Tuple(nombre: String, salario: Real)) =
    self.employees->collect(e | Tuple{nombre = e.name, salario = e.salary})
in estadisticas->forAll(t | t.salario > 0)

§6 Herencia y Design by Contract

6.1 Principio de Sustitución de Liskov

"Una subclase debe poder usarse en cualquier lugar donde se use la superclase sin violar las restricciones del sistema."

Implicaciones en OCL:

  • Los invariantes de la superclase se HEREDAN automáticamente: toda instancia de la subclase debe cumplir los invariantes de la superclase y los propios.
  • Las precondiciones de operaciones redefinidas solo pueden DEBILITARSE (or con la precondición padre).
  • Las postcondiciones de operaciones redefinidas solo pueden FORTALECERSE (and con la postcondición padre).

6.2 Invariantes e Herencia

context Stove
inv temperaturaMaxima: self.temperature <= 200

-- La subclase Oven hereda el invariante.
-- Puede agregar más restricciones:
context Oven
inv modoValido: self.mode = #bake or self.mode = #grill
-- Oven debe cumplir: temperature <= 200 AND (mode = bake or mode = grill)

6.3 Pre/Post condiciones e Herencia

Superclase Subclase
Precondición pre: condA pre: condA or condNueva (debilitada)
Postcondición post: condB post: condB and condNueva (fortalecida) 
-- Superclase
context Stove::open()
pre  apagarAntes: not self.isOn
post abierto:     self.isOpen = true

-- Subclase MicrowaveOven redefine open():
-- PRE debilitada (más permisiva):
context MicrowaveOven::open()
pre: not self.isOn or self.power = 0  -- permite abrir si potencia = 0

-- POST fortalecida (más restrictiva):
context MicrowaveOven::open()
post: self.isOpen = true and self.light = true  -- además enciende la luz

6.4 oclAsType para acceder a la superclase

-- Si Employee redefine 'age' pero queremos el age de Person:
context Employee
inv: self.oclAsType(Person).age >= 0

-- Verificar tipo antes de castear:
context Collection
inv: self->forAll(e |
    e.oclIsKindOf(Employee) implies
    e.oclAsType(Employee).salary > 0)

6.5 Ejemplo Completo (ClassicGuitar)

-- ClassicGuitar hereda de Guitar
-- Invariante: todas las cuerdas son de plástico
context ClassicGuitar
inv soloPlastico:
    strings->select(s | s.oclIsTypeOf(PlasticString))->size() = strings->size()

-- Equivalente usando forAll:
context ClassicGuitar
inv soloPlastico:
    strings->forAll(s | s.oclIsTypeOf(PlasticString))

§8 Tips: OCL ↔ Lenguaje Natural

Tabla de patrones para traducir entre lenguaje natural y OCL en ambas direcciones.

9.1 De Lenguaje Natural a OCL

Patrón en Lenguaje Natural OCL Notas
"Todo X debe cumplir Y" context X inv: <Y> Self es la instancia de X
"Todos los [col] deben…" col->forAll(x | ...) Cuantificador universal
"Al menos uno de [col] debe…" col->exists(x | ...) Cuantificador existencial
"Exactamente uno de [col]…" col->one(x | ...) Exactamente 1
"Ninguno de [col]…" col->forAll(x | not ...) o col->select(x|...)->isEmpty() Negación universal
"El número de [col] no puede superar N" col->size() <= N Cardinalidad máxima
"El número de [col] es exactamente N" col->size() = N Cardinalidad exacta
"Existe al menos un…" ...->exists(x | ...)
"El atributo se calcula como…" derive: <expr> Atributo derivado
"Al crear el objeto, el valor es…" init: <valor> Valor inicial
"Para ejecutar la op., se requiere…" pre: <condición> Precondición
"Al terminar la op., se garantiza…" post: <condición> Postcondición
"El resultado de la operación es…" post: result = <expr> result en postcondición
"El valor de X antes de ejecutar…" X@pre Solo en postcondiciones
"Los [col] que cumplen…" col->select(x | ...) Filtro
"Para cada X, calcular Y" col->collect(x | Y) Transformación
"La suma de todos los…" col->collect(...)->sum()
"No hay duplicados en…" col->isUnique(x | ...)
"X es de tipo exacto Y" x.oclIsTypeOf(Y) Tipo exacto
"X es de tipo Y (o subtipo)" x.oclIsKindOf(Y) Tipo o herencia
"Tratando X como Y" x.oclAsType(Y) Downcast explícito
"Para todas las instancias de la clase…" Clase.allInstances()->... Scope global
"Si A entonces B, si no C" if A then B else C endif Expresión condicional
"Sea variable = expr, entonces…" let var: T = expr in ... Variable local

9.2 De OCL a Lenguaje Natural

OCL Lenguaje Natural
context C inv nombre: self.attr >= 0 "Todo objeto C debe tener attr no negativo"
col->forAll(x | x.active) "Todos los elementos de col deben estar activos"
col->exists(x | x.priority = 'HIGH') "Debe existir al menos un elemento en col con prioridad HIGH"
col->one(x | x.isDefault) "Exactamente un elemento de col debe ser el predeterminado"
col->select(x | x.age > 18)->size() >= 2 "Debe haber al menos 2 elementos en col con edad mayor a 18"
col->collect(x | x.name)->asSet()->size() = col->size() "Todos los nombres en col son únicos"
self.salary@pre < self.salary "El salario aumentó respecto al valor anterior a la operación"
self.oclIsKindOf(Employee) implies self.salary > 0 "Si el objeto es un empleado (o subtipo), su salario debe ser positivo"
Persona.allInstances()->isUnique(p | p.dni) "No existen dos personas con el mismo DNI"
col->size() <= 5 "La colección no puede tener más de 5 elementos"
result = col->select(x | x.active)->size() "La operación devuelve el número de elementos activos en col"

9.3 Errores Comunes

Errores frecuentes
  1. Usar . en colecciones: self.employees.size() ❌ → self.employees->size() ✓
  2. Olvidar self: employees->size() puede funcionar dentro de context, pero es mejor explícito.
  3. collect vs select: collect transforma, select filtra. No confundir.
  4. @pre solo en post: atributo@pre solo es válido en postcondiciones.
  5. result solo en post/body: result solo en postcondiciones y body de operaciones.
  6. Set vs Bag: La navegación devuelve Set, collect() devuelve Bag. Si necesitas set: ->asSet().
  7. oclIsTypeOf vs oclIsKindOf: oclIsTypeOf(Dog) es false para un Poodle. Usar oclIsKindOf para incluir subtipos.

§9 Ejercicios Prácticos

Los siguientes ejercicios provienen del práctico de la materia. Intentá resolver cada ejercicio antes de ver la solución.

Ejercicio 1 — Modelo Personal / Empresa / Banco

Clases del modelo: Person (name, firstName, lastName, age, income(d:Date):Real, isMarried:Boolean, gender:Gender), Company (name, numberOfEmployers:Integer), BankAccount (balance:Real, accountNumber:String). Clases de asociación: Marriage (entre Person husband ↔ Person wife, date:Date), Job (entre Person employee ↔ Company employer, salary:Real, startDate:Date). Enumeración: Gender {male, female}.

Parte A — Traducir OCL a Lenguaje Natural OCL → Natural

(a) OCL → Natural inv

¿Qué expresa la siguiente expresión en lenguaje natural?

context Person
inv: self.isMarried implies self.age > 15

Si una persona está casada, entonces debe tener más de 15 años. Es decir, no se permiten matrimonios de personas de 15 años o menos.

(b) OCL → Natural inv

¿Qué expresa la siguiente expresión en lenguaje natural?

context Company
inv: self.numberOfEmployees = self.employee->size()

El atributo numberOfEmployees debe ser igual al número real de empleados de la empresa (tamaño de la colección employee).

(c) OCL → Natural pre post

¿Qué expresan las siguientes pre y post condiciones?

context Person::income(d: Date): Real
pre:  d > self.birthDate
post: if self.age < 18 then result < 100 else result < 200 endif

Precondición: la fecha consultada debe ser posterior a la fecha de nacimiento de la persona.

Postcondición: si la persona es menor de edad, su ingreso para esa fecha es menor a 100; si es mayor de edad, su ingreso es menor a 200.

(d) OCL → Natural inv

¿Qué expresan estas dos restricciones? ¿Cuál es la diferencia entre ellas?

-- Restricción 1
context Person
inv: BankAccount.allInstances()->collect(b | b.owner)->asSet()->size() = 3

-- Restricción 2
context Person
inv: BankAccount.allInstances()->collect(b | b.owner)->size() = 3

Restricción 1: El número de propietarios distintos de cuentas bancarias es exactamente 3. Se usa asSet() para eliminar duplicados de personas que tienen más de una cuenta.

Restricción 2: El número total de pares (propietario, cuenta) es exactamente 3, es decir, hay exactamente 3 cuentas bancarias en total (sin eliminar duplicados).

Diferencia: asSet() elimina duplicados, por lo que la restricción 1 cuenta personas únicas y la restricción 2 cuenta el total de cuentas.

(e) OCL → Natural inv

¿Qué expresa la siguiente restricción?

context Person
inv: self.Marriage[husband]->size() <= 1 and
     self.Marriage[wife]->size() <= 1

Una persona no puede ser marido en más de un matrimonio, y tampoco puede ser esposa en más de un matrimonio. Es decir, una persona solo puede estar casada una vez a la vez (en cada rol).

(f) OCL → Natural inv

¿Qué expresa la siguiente restricción?

context Person
inv: self.gender = Gender::male implies
         self.Marriage[wife]->isEmpty()

Si una persona es de género masculino, no puede aparecer en ningún matrimonio en el rol de "esposa". Es decir, los hombres solo pueden estar en el rol de "marido" en un matrimonio.

(g) OCL → Natural inv

¿Qué expresa la siguiente restricción?

context Person
inv: self.Job->collect(job | 
         Tuple{salario: Real = job.salary, 
               compania: String = job.employer.name})
     ->isUnique(t | t.compania)

Una persona no puede tener dos trabajos en empresas con el mismo nombre. Es decir, por cada empresa (identificada por nombre), una persona puede tener a lo sumo un trabajo.

Parte B — Escribir expresiones OCL Natural → OCL

(a) Natural → OCL inv

Expresar que el nombre y el apellido de una persona no puede tener más de 20 caracteres.

context Person
inv nombreCorto:   self.firstName.size() <= 20
inv apellidoCorto: self.lastName.size()  <= 20

Se pueden expresar como dos invariantes separadas o como una sola con and:

context Person
inv: self.firstName.size() <= 20 and self.lastName.size() <= 20

(b) Natural → OCL inv

Expresar que los gerentes (managers) de una empresa deben ser empleados de dicha empresa. Escribirlo de dos formas: con contexto en Person y con contexto en Company.

-- Opción 1: contexto en Person
context Person
inv: self.employer->forAll(c | c.manager = self implies
         self.employee->includes(self))

-- Opción 2: contexto en Company (más clara)
context Company
inv gerentes: self.manager->forAll(m | self.employee->includes(m))

(c) Natural → OCL inv

Expresar que toda empresa tiene al menos una empleada de género femenino.

context Company
inv empleadaFemenina:
    self.employee->exists(p | p.gender = Gender::female)

(d) Natural → OCL inv

Expresar que ninguna persona puede tener más de 5 cuentas bancarias.

context Person
inv maxCuentas: self.bankAccount->size() <= 5

(e) Natural → OCL inv

Expresar que ningún cliente (persona con al menos una cuenta bancaria) puede tener más de 5 cuentas.

context Person
inv maxCuentasCliente:
    self.bankAccount->notEmpty() implies self.bankAccount->size() <= 5

La diferencia con (d) es que aquí solo aplica la restricción a quienes ya tienen al menos una cuenta. Con (d) aplica a todas las personas. En la práctica, si una persona sin cuentas no puede tener más de 5, la restricción (d) es equivalente.

(f) Natural → OCL inv

Expresar que ninguna persona tiene dos trabajos en empresas con el mismo nombre.

context Person
inv sinDuplicadosEmpresa:
    self.Job->isUnique(j | j.employer.name)

También es válido con forAll:

context Person
inv: self.Job->forAll(j1, j2 |
         j1 <> j2 implies j1.employer.name <> j2.employer.name)

Clases del modelo: Persona (dni:String, tipoDoc:TipoDocumento, nombre:String, apellido:String, genero:Genero, fechaNac:Fecha, edad:Integer[derived], domicilio:String, estudios:NivelEstudios), Afiliado extends Persona (numAfiliado:Integer, fechaAfiliacion:Fecha, activo:Boolean), Partido (nombre:String, ambito:Ambito, presidente:Afiliado, afiliados:Set(Afiliado), electores:Set(Persona)), Mesa (numero:Integer, maximo:Integer[static,init:100], electores:Set(Persona)), Circuito (nombre:String, mesas:Set(Mesa)). (1) init

Parte 2 — Atributo derivado (derive)

Parte 4 — Invariantes

(a) Natural → OCL inv

El tipo de documento de una persona debe ser coherente con su género: las mujeres no pueden tener LE (Libreta de Enrolamiento).

context Persona
inv documentoCoherente:
    self.genero = Genero::femenino implies
        self.tipoDoc <> TipoDocumento::LE

(b) Natural → OCL inv

No se permiten matrimonios registrados después del año 2010. (La clase Matrimonio tiene fechaMatrimonio: Fecha; asumir que Fecha tiene operación anio(): Integer.)

context Matrimonio
inv matrimonioAnterior2010: self.fechaMatrimonio.anio() <= 2010

(c) Natural → OCL inv

El presidente de un partido debe tener estudios universitarios.

context Partido
inv presidenteUniversitario:
    self.presidente.estudios = NivelEstudios::universitario

(d) Natural → OCL inv

Para votar, una persona debe tener al menos 16 años.

context Mesa
inv electorMayorEdad:
    self.electores->forAll(p | p.edad >= 16)

(e) Natural → OCL inv

Una mesa no puede tener más electores que su máximo permitido.

context Mesa
inv limiteElectores: self.electores->size() <= self.maximo

(f) Natural → OCL inv

El domicilio de cada elector de una mesa debe estar dentro del circuito al que pertenece esa mesa. (Simplificación: el circuito tiene una operación contieneElector(p: Persona): Boolean.)

context Circuito
inv electorEnCircuito:
    self.mesas->forAll(m |
        m.electores->forAll(p | self.contieneElector(p)))

(g) Natural → OCL inv

El número de afiliado de cada afiliado debe ser único dentro del partido.

context Partido
inv numerosUnicos:
    self.afiliados->isUnique(a | a.numAfiliado)

(h) inv — Herencia / Principio de Liskov

Analizá: si Afiliado es subclase de Persona, ¿qué relación tienen sus invariantes? ¿Puede Afiliado tener un invariante que contradiga uno de Persona?

No puede contradecirlo. Por el principio de sustitución de Liskov, los invariantes se heredan: toda instancia de Afiliado debe cumplir los invariantes de Persona y los propios de Afiliado. Un invariante de Afiliado puede agregar restricciones pero nunca relajarlas ni contradecirlas.

Ejemplo: si Persona tiene inv: self.edad >= 0, Afiliado puede agregar inv: self.edad >= 18 (más restrictivo), pero no puede poner inv: self.edad >= 0 or self.edad = -1 (que sería más permisivo).

Parte 5 — Design by Contract (pre/post condiciones)

(a) Natural → OCL pre post

Operación Partido::afiliar(p: Persona). Afilia a una persona al partido.

  • Pre: la persona no está afiliada y tiene al menos 16 años.
  • Post: el conjunto de afiliados aumentó en uno con el nuevo afiliado.
context Partido::afiliar(p: Persona)
pre  noAfiliado:    not self.afiliados->exists(a | a.oclAsType(Persona) = p)
pre  personaValida: p.edad >= 16

post afiliadoIncluido:
    self.afiliados->size() = self.afiliados@pre->size() + 1
post nuevoAfiliado:
    self.afiliados->exists(a | a.oclAsType(Persona) = p)

(b) Natural → OCL pre post

Operación Partido::desafiliar(a: Afiliado). Desafilia a un afiliado (lo marca como inactivo).

  • Pre: el afiliado existe y está activo.
  • Post: el afiliado queda inactivo; el resto no cambia.
context Partido::desafiliar(a: Afiliado)
pre  esAfiliado: self.afiliados->includes(a)
pre  estaActivo: a.activo = true

post inactivo:        a.activo = false
post mismoTamanio:    self.afiliados->size() = self.afiliados@pre->size()
post otrosSinCambios:
    self.afiliados->select(x | x <> a)->forAll(x | x.activo = x.activo@pre)

(c) Natural → OCL pre post

Operación Partido::cambiarPresidencia(nuevo: Afiliado). Cambia el presidente del partido.

  • Pre: el nuevo presidente es afiliado activo y tiene estudios universitarios.
  • Post: el presidente es el nuevo afiliado.
context Partido::cambiarPresidencia(nuevo: Afiliado)
pre  esAfiliado:   self.afiliados->includes(nuevo)
pre  estaActivo:   nuevo.activo = true
pre  tieneEstudios: nuevo.estudios = NivelEstudios::universitario

post nuevoPresidente:  self.presidente = nuevo
post sinOtrosCambios:  self.afiliados = self.afiliados@pre

(d) Natural → OCL pre post

Operación Circuito::inscribirElector(m: Mesa, p: Persona). Inscribe un elector en una mesa del circuito.

  • Pre: la mesa pertenece al circuito, la persona no está ya inscripta en esa mesa, y la mesa no está llena.
  • Post: la persona es parte de los electores de la mesa.
context Circuito::inscribirElector(m: Mesa, p: Persona)
pre  mesaEnCircuito:  self.mesas->includes(m)
pre  noInscripto:     not m.electores->includes(p)
pre  mesaDisponible:  m.electores->size() < m.maximo

post inscripto:      m.electores->includes(p)
post tamanioCrecio:  m.electores->size() = m.electores@pre->size() + 1
